ODW — Read-only Client

A pure observatory: runs are launched by your agent via the CLI; the app only shows what's there and what's happening. Two modules — Workspace (read source) and Jobs (watch the live DAG) — plus a proposed third, Activity. No Run / Install / Hub.

Scope — converged to read-only

What's in

  • Workspace — local workflows (managed dir), each one's phases, structure diagram, and source. Read-only.
  • Jobs — every run (live + history), grouped to its workflow, with the live DAG, Logs, and Result.
  • Activity (proposed 3rd) — fleet load per adapter + a live firehose of the 9 event types.
  • Settings — adapters + app prefs only.

What's out (by design)

  • No Run / Launch / Execute — runs always start from the agent via odw run <name>.
  • No Install / Hub — distribution isn't in this iteration.
  • No param forms — the GUI never fills args to start a run.

Import into Figma (no plan upgrade)

The Figma MCP is capped at 6 calls/month on the Starter/View seat. The html.to.design plugin converts these screens into editable Figma layers:

  1. Serve locally: cd docs && python3 -m http.server 4319 → open http://127.0.0.1:4319/odw-client.html
  2. In Figma: Plugins → html.to.design, and install its browser extension.
  3. Open a screen above (each ?only=s0X URL is one clean frame) → click the extension → Send to Figma. Repeat per screen.
  4. Or just drag the 7 PNGs in docs/odw-client-screens/ onto a Figma canvas as reference frames.
