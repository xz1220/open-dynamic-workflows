Portable workflow scripts

Keep the plan in code. Let agents do the parallel work.

ODW runs the same JavaScript workflow dialect used by Claude Code: ordinary control flow, injected primitives, background execution, and CLI adapters that keep orchestration outside the host context.

agent(prompt)

Dispatch real work

Send a scoped task to a coding-agent CLI and collect text or schema-validated output.

parallel(thunks)

Fan out safely

Run independent subtasks concurrently when the next step needs the full batch.

pipeline(items)

Stream stages

Move each item through multi-step review, verification, or synthesis without a global barrier.

Runtime shape

A small engine between workflow scripts and real agent CLIs.

The runtime loads the workflow, injects primitives, schedules agent calls, and persists run state so long-running orchestration can be observed after it starts.

01

Load

Read a Claude Code-style script and wrap top-level return and await into a runnable function.

02

Schedule

Apply async concurrency limits and route each agent call through the selected adapter.

03

Persist

Store run state, logs, events, and results in a run directory outside the caller process.

04

Observe

Use the CLI to follow logs, check status, pause, resume, stop, or print the final result.

Start from source

Clone the runtime and run the test-backed CLI locally.

Open Dynamic Workflows is currently in an active TypeScript rewrite. The repository includes the CLI shell, runtime layers, mock-adapter tests, examples, and the skill documentation for host agents.
